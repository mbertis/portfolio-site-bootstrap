Meet App

The goal of this project was to build a serverless, progressive web application (PWA) with React using a test-driven development (TDD) technique. The application uses the Google Calendar API to fetch upcoming events. I started this project by developing user stories for each of the features I would be implementing in this app. I then turned these into Given-When-Then statements that formed the basis of my TDD approach. I really enjoyed continuing my experience with React and also using serverless functions.

myFlix API

The myFlix API is the server-side component of a "movies" web application which allows users to access information about different movies, directors, and genres. Users can also create and update their account information and create a list of their favorite movies. This RESTful API is a Node.js and Express application and uses URL endpoints to make Axios requests that are detailed in my documentation. The movies database was built using MongoDB and information is provided in JSON format. My API has been tested in Postman and is deployed to Heroku.

myFlix App - React

Using React, I created the client-side component of my myFlix application. This application is a single-page application (SPA) and uses state routing to navigate between views and share URLs. Parcel was initially used as its build tool. This app was written with React Redux and utilizes Bootstrap as a UI library for styling and responsiveness.
